# 第二次作业报告 - 拼音输入法

## 一、作业概述

本次作业实现了一个基于N-gram语言模型的拼音输入法系统，包括：
1. 数据预处理模块
2. N-gram语言模型
3. 拼音到汉字的转换算法
4. 图形用户界面（GUI）

## 二、实现细节

### 2.1 数据预处理 (data_preprocessing.py)

**功能：**
- 从网盘下载说明
- 数据解压
- 文本清洗（去除URL、邮箱、特殊字符）
- 训练集准备

**清洗规则：**
- 去除URL链接
- 去除邮箱地址
- 保留中文字符、英文字母、数字和常见标点
- 合并连续空格

### 2.2 语言模型 (language_model.py)

**模型类型：** 基于字的Bigram（2-gram）模型

**核心算法：**
- **N-gram统计：** 统计相邻字符的共现频率
- **加一平滑（Laplace Smoothing）：** 处理未登录词
  ```
  P(char | context) = (count + 1) / (context_count + vocab_size)
  ```

**主要方法：**
- `train()`: 训练模型，统计n-gram频率
- `get_probability()`: 计算条件概率
- `predict_next()`: 预测下一个最可能的字
- `save()/load()`: 模型持久化

### 2.3 拼音输入法 (pinyin_input.py)

**核心组件：**

1. **拼音字典：** 包含400+常用拼音到汉字的映射

2. **拼音切分算法：**
   - 支持空格分隔的拼音输入
   - 贪心算法切分连续拼音（如 "zhongwen" → ["zhong", "wen"]）

3. **Beam Search解码：**
   - 保留top-k个最优候选路径
   - 使用语言模型概率进行路径评分
   - 动态规划逐步扩展路径

**算法流程：**
```
输入: 拼音字符串
1. 切分拼音 → [p1, p2, ..., pn]
2. 对每个pi，获取候选汉字列表
3. 使用Beam Search找到最优汉字序列
   - 初始化: 第一个拼音的所有候选字
   - 迭代: 对每个后续拼音，扩展路径并保留top-k
   - 评分: 使用语言模型计算路径概率
4. 返回: 排序后的候选结果
```

### 2.4 图形界面 (gui.py)

**技术：** Tkinter

## 四、算法示例

### 示例1: 简单拼音转换
```
输入: zhongwen
拼音切分: ["zhong", "wen"]
候选汉字:
  - zhong: [中, 种, 重, 众, ...]
  - wen: [文, 问, 闻, ...]

Beam Search过程:
  初始: [中], [种], [重], ...
  扩展: [中文], [中问], [种文], [种问], ...
  评分: P(文|中) > P(问|中) > ...
  
输出: 中文 (最高概率)
```

### 示例2: 语言模型概率计算
```
句子: "今天天气很好"

Bigram概率:
  P(今|<BOS>) × P(天|今) × P(气|天) × P(很|气) × P(好|很) × P(<EOS>|好)

使用加一平滑后，即使训练数据中未出现的组合也有非零概率。
```
